Classes

EllipticalMercatorProjection

This is the Mercator projection assuming the earth is a perfect elipsoid. It's more exact, but the math is slightly more complex and thus slower.

EuclidianCoordinate

This class specifically models two-dimensional points in a flat plane

GlobalMercatorProjection

This is the base class of all Mercator projections that map the globe as a whole.

GlobalMesh

This class overlays the globe with a mesh of squares. The algorithm is based on subdividing the UTM grids into finer cell structures, so the coverage is for latitudes between 80° South and 84° North.

MercatorProjection

Base class for Mercator projections tranlating longitude/longitude values on the globe into X/Y coordinates on a flat map.

SphericalMercatorProjection

This is the Mercator projection assuming the earth is a perfect sphere. It's less exact, but the math is slightly easier and thus faster.

UtmCoordinate

UTM Coordinates need additionally the zone and the band info to identify the grid to which the X/Y values belong

UtmProjection

The Universal Transverse Mercator Projection

Structs

Angle

Encapsulation of an Angle. Angles are constructed and serialized in degrees for human convenience, but a conversion to radians is provided for mathematical calculations. Angle comparisons are performed in absolute terms - no "wrapping" occurs. In other words, 360 degress != 0 degrees.

Ellipsoid

Encapsulation of an ellipsoid, and declaration of common reference ellipsoids.

GeodeticCalculator

Implementation of Thaddeus Vincenty's algorithms to solve the direct and inverse geodetic problems. For more information, see Vincent's original publication on the NOAA website: See http://www.ngs.noaa.gov/PUBS_LIB/inverse.pdf

GeodeticCurve

This is the outcome of a geodetic calculation. It represents the path and ellipsoidal distance between two GlobalCoordinates for a specified reference ellipsoid.

GeodeticMeasurement

This is the outcome of a three dimensional geodetic calculation. It represents the path a between two GlobalPositions for a specified reference ellipsoid.

GlobalCoordinates

Encapsulation of latitude and longitude coordinates on a globe. Negative latitude is southern hemisphere. Negative longitude is western hemisphere. Any angle may be specified for longtiude and latitude, but all angles will be canonicalized such that: -90 <= latitude <= +90 -180 < longitude <= +180

GlobalPosition

Encapsulates a three dimensional location on a globe (GlobalCoordinates combined with an elevation in meters above a reference ellipsoid).

UtmGrid

The globe is partioned into Grids by the UTM projection. This structure represents such a grid.
